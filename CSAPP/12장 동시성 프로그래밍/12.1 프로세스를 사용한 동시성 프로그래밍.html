12.1 프로세스를 사용한 동시성 프로그래밍

동시성 프로그래밍의 기본 개념과 구현 방법을 설명하고 있습니다. 여기서는 특히 다중 클라이언트를 서버에 연결하고, 데이터를 전송하는 예시를 통해 프로세스와 포크(fork), 그리고 소켓(socket) 프로그래밍에 대해 설명합니다. 이해를 돕기 위해 각 그림의 내용을 단계별로 간단하고 명확하게 설명하겠습니다.

그림 12.1: 프로세스를 사용한 기본 서버 클라이언트 모델

  • 서버 (Server): 서비스를 제공하는 주체로, 클라이언트의 연결 요청을 기다립니다 (listenfd(3)).
  • 클라이언트 (Client 1, Client 2): 서비스를 요청하는 주체로, 서버에 연결을 요청합니다.
  • 연결 요청 (Connection request): 클라이언트가 서버에 접속을 요구할 때 사용하는 신호입니다.
  • 연결 수락 (Connection accept): 서버가 클라이언트의 연결 요청을 받아들이고, 데이터 전송을 위한 준비를 합니다 (connfd(4)).

그림 12.2 및 12.4: 서버가 데이터를 클라이언트에 전송


  • 데이터 전송 (Data transfers): 서버와 클라이언트 사이의 데이터 교환이 이루어집니다.
  • 포크(fork): 서버가 자신의 복사본을 생성하여 동시에 여러 클라이언트의 요청을 처리할 수 있습니다. 각 포크된 프로세스는 독립적으로 클라이언트와 통신합니다.

그림 12.3: 서버가 동시에 두 클라이언트의 연결을 처리

  • 동시 연결 처리: 서버가 동시에 여러 클라이언트의 요청을 받아들여 각각의 클라이언트에게 서비스를 제공할 수 있습니다.

주요 용어 설명

  • 프로세스(Process): 실행 중인 프로그램의 인스턴스입니다.
  • 포크(Fork): 프로세스가 자신의 복사본을 만드는 행위입니다. 이를 통해 부모 프로세스와 자식 프로세스가 생성됩니다.
  • 소켓(Socket): 네트워크를 통해 데이터를 교환하기 위한 엔드포인트입니다. IP 주소와 포트 번호로 정의됩니다.
  • 리슨(Listen): 서버가 특정 포트에서 클라이언트의 연결 요청을 기다리는 상태입니다.
  • 연결 디스크립터(Connection Descriptor): 연결된 클라이언트와 통신을 관리하는 핸들입니다.

이렇게 각 단계를 통해 서버가 어떻게 다중 클라이언트의 요청을 동시에 처리할 수 있는지를 보여주는 예제입니다. 이러한 개념은 웹 서버나 데이터베이스 서버 같은 실제 시스템에서 광범위하게 적용됩니다.
