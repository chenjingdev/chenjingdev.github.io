3.7.1 런타임 스택

스택은 컴퓨터 메모리의 중요한 영역 중 하나로 함수가 호출될 때마다 사용되는 데이터를 저장하는 공간입니다.

함수가 다른 함수를 호출할 때마다, 스택은 그 함수의 실행에 필요한 정보를 저장하기 위해 사용됩니다. 이 정보에는 로컬 변수, 함수 인자, 반환 주소 등이 포함됩니다.

  • LIFO 구조: 스택은 '마지막에 들어간 것이 먼저 나온다'(Last-In, First-Out, LIFO)의 특성을 갖고 있습니다. 이는 함수의 호출과 반환 과정을 생각해 볼 때 매우 자연스러운 동작입니다.
  • 프레임: 각 함수 호출 시 생성되는 스택의 영역을 '프레임'이라고 합니다. 하나의 함수가 다른 함수를 호출하면, 새로운 프레임이 스택에 추가되고 함수가 반환될 때 해당 프레임은 스택에서 제거됩니다.
  • 스택 포인터: 스택의 현재 위치를 나타내는 것으로, 스택의 'top'을 가리킵니다.

스택의 동작 예시

예를 들어, 함수 P가 함수 Q를 호출한다고 가정해 보겠습니다.

  1. P 함수가 Q 함수를 호출하면, 스택에는 P 함수의 프레임이 이미 존재합니다.
  2. Q 함수를 위한 새로운 프레임이 스택에 추가됩니다. 이 프레임에는 Q 함수의 로컬 변수, 인자, 그리고 P로 돌아갈 때 필요한 반환 주소가 포함됩니다.
  3. Q 함수의 실행이 끝나고 반환하면, Q 함수의 프레임은 스택에서 제거되고, 스택 포인터는 다시 P 함수의 프레임을 가리키게 됩니다.

스택의 중요성

스택은 함수 호출과 관련된 정보를 관리하여 프로그램이 예상대로 실행되도록 돕습니다. 각각의 함수가 독립적인 실행 공간을 갖게 하며, 함수들이 서로의 변수를 침범하지 않도록 합니다.

제공된 이미지는 스택의 구조를 시각적으로 보여줍니다. 주소가 증가하는 방향으로 다음과 같은 구성 요소들이 있습니다.

  • 인자 생성 영역: 다른 함수를 호출할 때 인자를 전달하기 위한 영역입니다.
  • 로컬 변수: 함수의 로컬 변수들이 저장되는 공간입니다.
  • 저장된 레지스터: 함수가 다른 함수를 호출하기 전에 현재 함수의 상태를 보존하기 위해 필요한 레지스터들이 저장됩니다.
  • 반환 주소: 함수가 끝나고 제어가 반환될 주소가 저장됩니다.

이 스택 구조를 통해 프로그램은 여러 함수 호출을 효율적으로 관리하고 제어 흐름을 안정적으로 유지할 수 있습니다.

컴퓨터 시스템을 배우면서 알아야 할 핵심 용어들을 정리해 드리겠습니다.

용어 설명:

  • 스택(Stack): 함수의 호출과 반환에 사용되는 LIFO 구조의 메모리 영역.
  • 프레임(Frame): 개별 함수 호출에 필요한 정보를 담고 있는 스택의 하위 구조.
  • 스택 포인터(Stack Pointer): 현재 스택의 'top'을 가리키는 지시자.
  • 로컬 변수(Local Variables): 함수 내에서 선언되어 함수 실행 동안에만 존재하는 변수.
  • 저장된 레지스터(Saved Registers): 함수 호출 전에 현재 상태를 저장하기 위해 사용되는 레지스터들.
